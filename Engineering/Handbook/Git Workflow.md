# Commit Types

docs:

Documentation updates.

Example

docs: update Enterprise Memory release documentation

---

feat:

New platform capability.

Example

feat: implement Mission Timeline

---

fix:

Bug fixes.

Example

fix: resolve dropdown navigation issue

---

refactor:

Improve existing code without changing behavior.

---

test:

Testing improvements.

---

chore:

Maintenance tasks.

---

# Commit Philosophy

Every commit should represent one meaningful engineering change.

Avoid combining unrelated work into a single commit.
